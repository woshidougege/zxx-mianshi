🛠️ 测试技术详解

1. 功能测试

核心能力

  • 测试用例设计：等价类划分、边界值分析、因果图、判定表、场景法
  • 测试流程：需求评审、测试计划、测试方案、用例设计、用例执行、测试报告
  • 缺陷管理：缺陷提交、跟踪、验证、关闭

面试怎么说

"功能测试是测试工程师的基本功。我的方法是：先理解需求，参加需求评审；然后设计用例，用等价类、边界值这些方法覆盖各种场景；执行用例时会结合实际情况补充探索性测试。"

"我比较关注边界场景和异常场景，因为这些地方最容易出问题。比如输入框的长度限制、数值的上下限、并发操作、网络异常等。"

2. 接口测试

工具和方法

  • JMeter：接口功能测试、性能测试、自动化回归
  • Postman：接口调试、参数化测试、环境管理
  • 测试重点：参数校验、返回值校验、状态码、错误码、数据库校验

面试怎么说

"接口测试我主要用 JMeter 和 Postman。Postman 用来做接口调试和临时验证，JMeter 用来做批量测试和自动化回归。"

"接口测试重点关注几个方面：一是参数校验，必填参数、参数类型、参数长度等；二是返回值校验，状态码、数据格式、字段值是否正确；三是数据库校验，接口调用后数据库数据是否符合预期。"

3. 性能测试

测试类型

  • 压力测试：测试系统在高负载下的表现
  • 并发测试：测试系统同时处理多个请求的能力
  • 稳定性测试：测试系统长时间运行的稳定性

关注指标

  • QPS：每秒查询数（Queries Per Second）
  • RT：响应时间（Response Time）
  • 错误率：失败请求占比

面试怎么说

"性能测试我用 JMeter 做。首先要明确测试目标，比如这个接口要支持多少 QPS，响应时间控制在多少毫秒以内。然后设计测试场景，配置并发数、持续时间、压力模型。"

"压测过程中要实时监控系统指标，不只是看 JMeter 的报告，还要看服务器的 CPU、内存、数据库连接数等。"

4. 自动化测试

技术栈

  • Python + pytest + allure：接口自动化测试框架
  • JMeter：接口自动化回归
  • Jenkins：CI/CD 集成，定时执行

实践经验

  • 在水滴保项目中使用 Python + pytest + allure 搭建接口自动化框架
  • 在贝壳项目中通过 sosotest 维护核心接口自动化
  • 在 vipkid 项目中基于 JMeter + 数据库校验实现自动化回归

面试怎么说

"我做自动化测试主要是接口自动化，因为接口相对稳定，自动化的投入产出比比较高。"

"我的做法是：先梳理核心业务流程，把流程分解成接口调用链；然后用 pytest 编写测试用例，每个用例包括准备数据、调用接口、断言结果、清理数据；最后集成到 Jenkins，每次代码合并或发布前自动执行。"

Q7：自动化测试的价值是什么？

参考答案："自动化测试的价值主要体现在：

  1. 提升回归效率：以前手工回归要半天，自动化几分钟就能跑完
  2. 保证回归质量：自动化执行更稳定，不会漏测
  3. 快速反馈：集成到 CI/CD，每次发布前自动执行
  4. 释放人力：可以做更有价值的探索性测试

当然自动化需要维护成本，所以我一般对核心、稳定的业务做自动化。"

Q8：性能测试怎么做？

参考答案："性能测试的步骤是：

  1. 明确目标：要支持多少 QPS，响应时间多少
  2. 设计场景：配置并发数、持续时间、压力模型
  3. 执行压测：用 JMeter 执行，监控系统指标
  4. 分析结果：看 QPS、响应时间、错误率，定位瓶颈
  5. 优化验证：开发优化后再次压测，验证效果

Q12：发现bug开发不认可怎么办？

参考答案："首先我会确认这个确实是bug。然后和开发沟通，解释为什么这是bug，影响是什么。如果开发还是不认，我会找产品确认需求，或者找测试经理协调。大部分情况下通过沟通都能解决。"

总结

面试准备清单

  • ☑️ 熟记个人优势和核心项目
  • ☑️ 准备自我介绍（1-2分钟）
  • ☑️ 梳理每个项目的介绍话术
  • ☑️ 准备常见技术问题的回答
  • ☑️ 准备离职原因和职业规划
  • ☑️ 准备1-2个提问（了解团队、业务、技术栈）

面试心态

✅ 自信但不傲慢：相信自己的能力，但保持谦虚

✅ 真实不夸大：说自己真实做过的，不会的可以说"了解一些"

✅ 积极主动：展现学习能力和工作热情

✅ 沟通顺畅：语速适中，逻辑清晰，善于举例
